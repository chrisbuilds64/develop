## Development Principles

This project follows specific principles (see [CLAUDE.md](https://github.com/chrisbuilds64/control/blob/main/CLAUDE.md)):

- **KISS** - Keep It Simple, Stupid
- **YAGNI** - You Aren't Gonna Need It
- **Use Case Driven** - Real problems, real solutions
- **Ship > Perfect** - Working code beats perfect plans

## Bug Reports

**Good bug reports include:**
- Clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Your environment (OS, Python/Flutter version, etc.)
- Relevant logs or screenshots

## Feature Requests

**Good feature requests:**
- Describe the **problem** you're trying to solve
- Explain **why** this feature would be valuable
- Consider if it fits the project's scope (productivity apps)
- Accept that not every feature will be implemented (YAGNI!)
